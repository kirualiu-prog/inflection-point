# 臨界點解謎室 網頁遊戲計劃

## 1. 目標

把目前的分鏡圖與影片內容，落地成一款可直接在瀏覽器遊玩的敘事解謎遊戲。

核心方向：
- 主題保留「企業調查 / 財報異常 / 風險判讀」的科技懸疑感
- 玩法以「看資料、找線索、做判斷」為主
- 第一版先做成單機、可通關、可部署的 MVP

- 主要互動形式是選擇題與線索判讀
- 有倒數計時、風險警示、答對回饋、數值重算等節奏

## 3. 遊戲定位

### 玩法類型
- 單人敘事解謎
- 資料推理
- 互動式案件分析

### 玩家體驗
- 像在操作一個調查系統
- 先讀資料，再下判斷
- 選錯會看到錯誤方向的後果
- 選對會看到「真相揭露」與數值翻轉

### 第一版範圍
- 1 個首頁
- 1 個教學 / 任務引導頁
- 1 個核心關卡
- 1 個結果頁
- 1 個關卡完成狀態

## 4. 核心玩法

### 主要循環
1. 接收任務
2. 查看公司資料與異常訊號
3. 讀取線索卡
4. 選擇答案
5. 顯示正誤與原因
6. 重算結果
7. 進入下一段故事

### 第一關玩法重點
- 題目是「這家公司最大的風險是什麼？」
- 玩家要從財報、成本、碳排、營運異常中找真正原因
- 答案不是表面數字，而是被忽略的真實成本

### 建議機制
- 倒數計時
- 答題一次或有限次重試
- 提示按鈕
- 分數或評等
- 存檔進度

## 5. 關卡結構

### 關卡 1
- 目標：找出消失的獲利
- 核心：財報異常判讀
- 形式：單題推理 + 結果解說

### 關卡 2
- 目標：找出成本黑洞
- 核心：多來源線索交叉比對
- 形式：多選線索 + 排除法

### 關卡 3
- 目標：確認真正風險來源
- 核心：整合前兩關資訊
- 形式：最終判斷與結局

## 6. 頁面規劃

### A. 首頁
- 遊戲標題
- 開始任務按鈕
- 風格化背景與載入動畫
- 進度入口

### B. 任務頁
- 任務名稱
- 案件編號
- 系統狀態
- 任務說明
- 啟動任務按鈕

### C. 調查頁
- 中央報告卡
- 左側資料掃描狀態
- 右側風險警示
- 下方系統訊息
- 線索卡與題目區

### D. 答題結果頁
- 正確 / 錯誤狀態
- 原因解說
- 數值重算展示
- 下一步按鈕

### E. 關卡完成頁
- 通關結果
- 得分或評等
- 下一關解鎖
- 返回首頁

## 7. 視覺方向

### 風格
- 深色科技感
- 鍍金 / 橘紅警示色
- 青藍數據光效
- 金屬框線與資料面板

### UI 元件
- HUD 邊框
- 線索卡
- 倒數圓環
- 風險警報
- 系統訊息框
- 數值圖表

### 動效原則
- 少量、精準、有用途
- 開場載入
- 線索亮起
- 選項反饋
- 正確答案揭示
- 風險數值翻轉

## 8. 技術方案

### 前端
- `Vite + React`
- 以單頁應用方式實作
- 內容資料用 JSON 管理

### 狀態管理
- 先用 React state / context 就夠
- 不先上大型狀態庫

### 存檔
- `localStorage`
- 記錄已完成關卡、分數、提示使用狀態

### 動畫
- 先用 CSS
- 必要時再補少量動畫函式庫

### 音效
- 點擊
- 警報
- 答對
- 答錯
- 載入

## 9. 資料結構

建議把關卡寫成資料檔，方便後續擴充。

每關至少包含：
- `id`
- `title`
- `mission`
- `question`
- `choices`
- `correctAnswer`
- `explanation`
- `clues`
- `scoreRule`
- `timeLimit`
- `nextStage`

## 10. 開發里程碑

### Phase 1：原型
- 建首頁
- 建第一關主畫面
- 能選答案
- 能顯示對錯

### Phase 2：體驗完善
- 加載入動畫
- 加倒數計時
- 加答題結果頁
- 加基本音效

### Phase 3：內容擴充
- 補第二、第三關
- 整理題庫資料
- 加通關評分

### Phase 4：部署準備
- 清理資源引用
- 檢查手機版
- 做基本效能與錯誤檢查

### Phase 5：上線
- 推到 GitHub
- 連 Cloudflare 部署
- 驗證正式網址可正常打開

## 11. GitHub 與 Cloudflare 部署計劃

### GitHub
- 建立專案儲存庫
- 把遊戲原始碼推上去
- 保留清楚的版本紀錄
- 用分支管理後續新增關卡

### Cloudflare
- 用 Cloudflare Pages 部署前端
- 連接 GitHub 儲存庫
- 每次推送自動更新預覽版
- 主分支自動發佈正式版

### 部署檢查項目
- 首頁可正常載入
- 關卡可切換
- 圖片資源可正常顯示
- 手機版排版正常
- 404 與重新整理不會壞掉

## 12. 風險與決策

### 目前可先不做
- 後端帳號系統
- 多人連線
- 即時排行榜
- 複雜資料庫

### 原因
- 第一版目標是把玩法做出來，不是先做平台
- 這些功能之後都能加，但不該先拖慢上線

## 13. 成功標準

第一版完成時，應該能做到：
- 玩家從首頁進入遊戲
- 看懂任務目標
- 完成至少 1 關推理
- 看到正確答案與解釋
- 可以在 GitHub 與 Cloudflare 上正常開啟
